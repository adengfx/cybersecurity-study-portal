Connection Types

  • Wired (Ethernet): RJ-45 connector, also called CAT 5 / CAT 6 cables.
  • Wireless LAN (Wi-Fi): Limited geographic region.
  • Cellular: Wide coverage. Devices like phones can use both Wi-Fi and cellular simultaneously.

Benefits of Networking

  • Resource sharing — files, printers, storage.
  • Communications — email, VoIP (Voice over IP), video, instant messaging.
  • Redundancy — back up critical data over the network to another location.
  • Internet access — the biggest everyday benefit.
  • Device monitoring — alerts from IoT devices, security cameras, industrial equipment.
  • Centralised administration — manage multiple devices from one location without visiting each one.
CH 1

Network Addresses

MAC Addresses – Physical Addressing

  • Assigned by the manufacturer — called a burned-in address. Stored on the NIC.
  • 48 bits total, split into two 24-bit halves.
  • First 24 bits = OUI (Organisationally Unique Identifier) — identifies the manufacturer/vendor.
  • Last 24 bits = device-specific identifier assigned by the manufacturer.
  • No two devices in the world should share the same MAC address.
  • Switches use MAC addresses to make forwarding decisions (Layer 2).

IPv4 Addresses – Logical Addressing

  • A logical address — assigned to the device, not burned in at manufacture.
  • 32 bits, written in dotted decimal notation: four octets (8 bits each) separated by dots. E.g. 192.168.1.100
  • Divided into a network portion and host portion by a subnet mask.
  • Private IP ranges — usable inside networks but not routable on the public internet (e.g. 10.x.x.x, 192.168.x.x).
  • IPv4 addresses are exhausted — workaround is NAT.

IPv6 Addresses

  • 128 bits — gives 2128 possible addresses. Solves IPv4 exhaustion entirely.
  • Written in hexadecimal notation — digits 0–9 and letters A–F.
  • Structure: 8 quartets (groups of 4 hex digits) separated by colons.
  • Network portion = prefix. Host portion = host. Prefix length shown as e.g. /64 (no subnet mask — prefix length replaces it).
  • A /64 prefix means 64 bits for the network, 64 bits for the host.
CH 2

Places and Parts of a Network

Network Interface Card (NIC)

  • Circuitry that allows a device to send and receive data on wired or wireless networks.
  • Often built into the motherboard. Can be added as internal (PCIe slot) or external (USB).
  • Each NIC has a unique MAC address burned in at manufacture.

Switches

  • Make forwarding decisions based on MAC addresses — operate at Layer 2 (Data Link).
  • Maintain a MAC address table mapping device MAC addresses to switch ports.
  • All devices on a switch belong to the same IP network by default.
  • Data units at Layer 2 are called frames.

Routers

  • Interconnect different networks. Make forwarding decisions based on IP addresses — operate at Layer 3 (Network).
  • Maintain an IP routing table. Directly connected networks added automatically on startup.
  • Default route (0.0.0.0/0) — used when no specific route is known; points to the internet. Contains all possible IPv4 addresses.
  • Longest prefix match — when multiple routes match a destination, the router picks the most specific (most bits in subnet mask).
  • Routing protocols (e.g. OSPF) allow multiple routers to share routing information.
  • Data units at Layer 3 are called packets.

Wireless Access Points (APs)

  • Provide wireless connectivity. Connect back to the wired network via an ethernet switch.
  • Multiple APs allow roaming — client automatically connects to the AP with the strongest signal.
  • Home wireless routers combine AP + router + switch in one device.

Copper Cabling (Twisted Pair)

  • Wires twisted in pairs to combat electromagnetic interference (EMI).
  • UTP — Unshielded Twisted Pair. Standard.
  • STP — Shielded Twisted Pair. Extra foil/metal braiding for high-interference environments.
  • Plenum-rated cable — required in plenum spaces (HVAC airflow areas above drop ceilings / below raised floors). Standard cable releases toxic fumes when burned.
  • Connectors: RJ-45 (8 pins/conductors). Wiring standards: T568A and T568B (only difference: orange and green pairs swapped). Be consistent — don't mix.
  • Maximum distance: 100 metres for most categories.

Fibre Optic Cabling

  • Uses light (laser) to transmit data — zero EMI, longer distances than copper.
  • Structure: core (inner glass) + cladding (outer glass, different refractive index). Light is kept inside by total internal reflection.
  • SMF (Single-Mode Fibre) — very small core. Only one light path (mode). No multimode delay distortion. Used for long distances.
  • MMF (Multi-Mode Fibre) — larger core. Multiple light paths. Susceptible to multimode delay distortion (different modes arrive at different times, bits can overtake each other). Shorter distances.

PDU: Frames

Layer 1

PDU: Bits

  • Bottom-up mnemonic: Please Do Not Throw Sausage Pizza Away
  • Top-down mnemonic: All People Seem To Need Data Processing
  • PDU names bottom-up: Bacon Frying Produces Salivation (Bits · Frames · Packets · Segments)
  • OSI is a reference model, not a strict rule — not every component fits neatly into one layer.

TCP/IP Model

  • Combines OSI layers 5, 6, 7 into a single Application layer.
  • Combines OSI layers 1 and 2 into a Network Access layer (also called Network Interface or Link layer).
  • OSI Layer 3 becomes the Internet layer. Layer 4 keeps the name Transport.
  • Standard TCP/IP = 4 layers. A 5-layer variant keeps Physical and Data Link separate.

TCP vs UDP

  • TCP (Transmission Control Protocol) — reliable. Confirms data was received. Used where accuracy matters (web browsing, email, file transfer).
  • UDP (User Datagram Protocol) — unreliable. Sends and hopes it arrives. No confirmation. Used where speed matters (VoIP, video streaming, DNS).

Network Services

DHCP – Dynamic Host Configuration Protocol

  • Automatically assigns IP address information to devices — eliminates manual configuration.
  • Process remembered as DORA:
D
Discover
Client broadcasts asking if any DHCP server exists. Doesn't cross routers.
O
Offer
DHCP server responds with an offer. Client selects the first offer received.
R
Request
Client sends a request to the chosen server asking for IP address info.
A
Acknowledge
Server sends IP address, subnet mask, default gateway, and DNS server address.
  • If a router separates client from server, the router must be configured as a DHCP relay agent to forward discover broadcasts.
  • DHCPv6 Stateful: Server provides full IPv6 address (prefix + host) and optional info.
  • DHCPv6 Stateless: Router provides prefix via NDP (Neighbour Discovery Protocol). Client self-generates host portion using EUI-64 (derived from 48-bit MAC address). Stateless DHCPv6 provides extras like DNS server address.

DNS – Domain Name System

  • Translates human-readable names (e.g. linkedin.com) into IP addresses so packets can be routed.
  • Client sends a query to its DNS server (address learned from DHCP). If the local DNS server doesn't know, it queries another DNS server on the internet.
  • Uses TCP and UDP port 53.

NAT – Network Address Translation

  • Translates private (non-routable) IP addresses into publicly routable IP addresses at the router.
  • Millions of networks worldwide can reuse the same private IP ranges internally without conflict.
  • Inside Local — private IP of a device inside the network (not routable on internet).
  • Inside Global — publicly routable IP representing that same inside device.
  • Dynamic NAT — pool of public IPs. Each inside device gets a different public IP from the pool.
  • PAT (Port Address Translation) — multiple inside devices share a single public IP, distinguished by unique source port numbers. Also called NAT overload. Most common in home routers.
  • Router maintains a NAT translation table to track all mappings.

NTP – Network Time Protocol

  • Keeps device clocks synchronised. Critical for log accuracy during troubleshooting and for digital certificate validity checking.
  • Uses UDP port 123.
  • Stratum number — measures how many hops from an atomic clock. Lower = more accurate.
  • Stratum 0: Atomic clock (Caesium-133). Never directly accessible on the network.
  • Stratum 1: Server that learns directly from an atomic clock.
  • Stratum 2+: Increments by 1 per hop. Stratum 16 = invalid (too far from atomic clock).
  • US official time keeper: Naval Observatory (Washington D.C. and Colorado Springs).

QoS – Quality of Service

  • Best described as "managed unfairness" — intentionally treats some traffic better than others based on business need.
  • Only useful for periodic congestion — persistent congestion requires more bandwidth.
  • Causes of congestion: Speed mismatch (e.g. 1 Gbps in, 100 Mbps out) · Aggregation points (multiple fast inputs to one slower output).
  • Classification and marking: Tag traffic early with a priority marker so downstream devices can act on it without re-inspecting.
  • Queuing: Divide the single output queue into virtual sub-queues (e.g. VoIP queue vs best-effort queue). High-priority traffic won't be dropped even when best-effort queue is full.
CH 5

Wireless Networks

Types of Wireless LAN

  • Ad hoc: Two devices communicate directly — no infrastructure needed. E.g. AirDrop between iPhones. Uses Bluetooth or Wi-Fi. Does not scale.
  • Infrastructure: Clients connect to APs which connect to the wired network. Scalable. Supports roaming, monitoring, and security configuration (e.g. guest network isolation).
  • Mesh: APs don't need a wired connection — they receive and retransmit wireless signals. Needs power only. Ideal for areas without ethernet access (e.g. car parks, outdoor areas).

  • Channel bonding: Bonding multiple 20 MHz channels together increases throughput. 80 MHz = 4 channels; 160 MHz = 8 channels.

Wi-Fi Antenna Types

  • Omnidirectional (dipole): Transmits in all directions perpendicular to the antenna. Used in most home/office APs. Keep vertical for horizontal coverage. Multiple antennas enable MU-MIMO (Multi-User Multiple Input Multiple Output) — simultaneous connections to multiple clients.
  • Directional (parabolic): Focuses signal in one direction. Used to link two buildings — mount on each roof pointing at each other.

CH 6

Emerging Trends in Networking

SDN – Software Defined Networking

  • Traditional networking: configure one device at a time via SSH or GUI.
  • SDN: application expresses intent → SDN controller → pushes commands to all network devices at once.
  • Southbound Interfaces (SBI): Between SDN controller and network devices (south = below). Protocol: OpenFlow for SDN-aware devices; SSH for legacy devices.
  • Northbound Interfaces (NBI): Between applications and the SDN controller (north = above). Uses REST (Representational State Transfer) via HTTP verbs (GET, POST etc.).
  • Applications typically written in Python.
  • Data formatted using XML (Extensible Markup Language) or JSON (JavaScript Object Notation).

Virtualisation

  • Multiple virtual machines (VMs) run on a single physical server — each with its own OS, sharing underlying hardware.
  • Hypervisor — software that creates, starts, stops and monitors VMs.
  • Type 1 hypervisor: Installed directly on hardware, no underlying OS. E.g. VMware ESXi. More efficient.
  • Type 2 hypervisor: Installed on top of an existing OS. E.g. VMware Fusion. Easier to set up.
  • Containers: Share the same underlying OS (unlike VMs which each have their own). Smaller and more portable. E.g. Docker.
  • Network components can also be virtualised — virtual routers, virtual switches.

Cloud Technologies

  • Compute resources hosted by a cloud provider. No need to buy, house or maintain own hardware. Scale up/down on demand.
  • Major providers: AWS (Amazon Web Services) · Microsoft Azure.
  • SaaS — Software as a Service. Access applications via browser. E.g. Google Workspace.
  • IaaS — Infrastructure as a Service. Remote compute and storage.
  • PaaS — Platform as a Service. Virtual platforms for software developers.
  • Public cloud: Shared resources owned by a provider, multiple customers.
  • Private cloud: Dedicated resources for one organisation. Can be on-premises or hosted.
  • Hybrid cloud: Combination of public and private cloud.
  • Connect to cloud via internet + VPN (Virtual Private Network) — encrypted tunnel to protect data in transit.

5G and Wi-Fi 6

  • 5G — fifth generation cellular. Frequencies: 30–300 GHz (millimetre wave). Theoretical max: 20 Gbps. Typical real-world: ~1.5 Gbps. Latency as low as 1 millisecond.
  • 5G signals easily absorbed by walls/windows — requires dense base station deployment (street lamps, traffic lights). Cannot simply reuse existing 4G towers.
  • Wi-Fi 6 (802.11ax): Standardised 2019. Theoretical max 9.6 Gbps. Key: 1024-QAM · 160 MHz bonding · OFDMA · Target Wake Time · BSS Colouring · 8 simultaneous streams.
