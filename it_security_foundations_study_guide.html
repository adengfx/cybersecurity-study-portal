CH 1

Core Security Principles

The CIA Triad

A widely recognised information assurance model. The goal is to keep data private, unchanged, and available.

C
Confidentiality
Keeping private information private by preventing unauthorised access.
Violation: A hacker reading emails on a government mail server, or someone accessing a patient's medical file without authorisation.
I
Integrity
Protecting data from unauthorised change — either maliciously or accidentally.
Violation: A student changing their algebra grade from a C to an A, or someone altering an account balance.
A
Availability
Ensuring data and services are accessible to authorised users when needed.
Threats: DDoS attacks (flood systems to lock out legitimate users using botnets) and ransomware (holds data hostage until ransom is paid).

Protecting the CIA Triad

GoalMethod
ConfidentialityVerify identity (passwords/PINs, smart cards, fingerprints, passkeys, authenticator apps). Assign permissions so only authorised users access data. Use encryption to scramble data — readable only with a key.
IntegrityUse software that monitors for suspicious activity and notifies if unauthorised changes are made to data.
AvailabilityKeep systems current and patched. Back up regularly and store data in an offsite location.

Risk, Threats and Vulnerabilities

  • Risk — when a person, place, or thing is open or exposed to harm. Formula: Risk = Threats × Vulnerabilities
  • Threat — anything that can exploit a vulnerability, intentionally or accidentally, to obtain, damage, or destroy an asset. Threats are generally difficult to control.
  • Vulnerability — a security flaw or weakness in a system that can be exploited. Most vulnerabilities can be fixed.
  • Assets — tangible (computers, printers) or intangible (databases, trade secrets, company records).
  • If there is no vulnerability, there is no risk. Reducing vulnerabilities reduces overall risk.

Social Engineering

  • Cybercriminals use social engineering to trick people into revealing information or completing an action they wouldn't normally do.
  • Methods: phishing (email), vishing (phone), instant messaging/SMS, dumpster diving, shoulder surfing, and simple persuasion.
  • Dangerous because it requires no technical skill — simple persuasion is enough to breach a system.
  • Scareware — a spoofed ransomware message. Appears as a pop-up or redirect claiming the user has a virus, designed to trick them into clicking a malicious link or contacting a fake support number.
  • Best practices: use strong passwords/passphrases, don't trust unsolicited contacts online, don't share card details over IM or email, disable push notifications.
CH 2

Understanding Malware

Types of Malware

TypeDescription
VirusSelf-replicates but needs a transport agent (e.g. email attachment, flash drive, downloaded file) to spread to another host.
WormSelf-propagating — spreads independently without a transport agent. Consumes memory, processing, and bandwidth.
RansomwareHolds data hostage until a ransom is paid. Can block system access, encrypt files, stop applications, or destroy files if unpaid.
SpywareRecords keystrokes and activity (browsing habits, passwords, email accounts) and sends the data to a remote collection site, usually without the user's knowledge.
BotMalware created to complete a specific task. Usually part of a botnet (an army of bots) controlled by a bot master that triggers attacks at a predefined time.
RootkitA collection of programs that infiltrates a system, creates a backdoor, and remains undetected — allowing a hacker to take administrator-level control.
AdwareTracks browsing activity and sends advertising data to a collection site. Often bundled with free software.

OS Defences Against Malware

OSBuilt-in Tools
macOS Constantly monitors the system. Blocks malware from executing. Prevents unauthorised app access to critical files. Tools: Notarization and XProtect — signatures updated daily, prompts user to remove detected malware.
Linux ClamAV (cross-platform open-source antivirus), LMD (Linux Malware Detect — best combined with ClamAV). For rootkits: Rootkit Hunter and chkrootkit.
Windows Microsoft Defender — activates automatically if no other antimalware is detected. DEP (Data Execution Prevention) — prevents malware executing code from memory reserved for Windows. Microsoft Safety Scanner — free downloadable tool to remove malware (not a replacement for antimalware).
CH 3

Breaching Your System

Trojans

  • A program that appears innocent but is designed to cause malicious activity or provide a backdoor to a system.
  • Uses a wrapper — the malware is hidden inside something enticing, such as a game, utility, or message.
  • Modern Trojans are modular — can steal passwords, embed rootkits, or launch ransomware.
  • Avoid executable files from unknown sources: .exe .bat .vbs .scr
  • Best practices: avoid risky websites, don't download free system optimisers or games from unknown sources, use antimalware that updates automatically.

Rootkits and Logic Bombs

  • Rootkits can remain undetected for months or even years.
  • A rootkit is not a virus — it may have no identifiable signature, allowing it to evade antimalware detection.
  • Can gather intelligence by monitoring email, chats, passwords, and data transactions, then send that information to a remote server.
  • A logic bomb can be left behind by a rootkit — triggers an attack at a predefined time.
  • How rootkits get in: drive-by downloads, phishing with a macro-enabled Word doc, stolen credentials, or embedded on a flash drive.
  • Best practices: strong passwords on admin accounts, caution with email attachments, patch and update OS and applications.

Zero-Day Attacks

  • A zero-day vulnerability is a flaw in software that is unknown to the vendor.
  • A zero-day attack occurs when a hacker exploits the security hole before the vendor becomes aware of it.
  • Antimalware cannot detect it — there is no signature to use and compare.
  • Once the vendor discovers the vulnerability, they develop and release a patch — but systems remain at risk until the patch is applied.
  • Protection: use anomaly/profile-based detection (monitors malware-like behaviour rather than signatures), keep browser updated, think before you click.

IoT Threats

  • IoT devices collect and exchange data but have large attack surfaces — entry points including weak/missing passwords, outdated software, and physical access.
  • Attack vectors include gaining access to wireless signals or open ports.
  • Manufacturer responsibilities: continuous security updates, password protection, dedicated vulnerability reporting, timely resolution of weaknesses.
  • Research devices before purchase and keep firmware updated.

Internet Security

Cloud Computing Models

IaaS
Infrastructure as a Service
Servers, storage, network firewalls, data centre facilities.
PaaS
Platform as a Service
Development tools, database management, business analytics, OS environment.
SaaS
Software as a Service
Hosted applications. E.g. Microsoft Office 365.

Secure Connections and HTTPS

  • HTTPS uses TLS to encrypt data between you and the website — both sides exchange a shared key before any information is transferred.
  • HTTPS does not mean a website is safe — only use sites you know and trust.
  • Best practices: check for HTTPS in the address bar, verify contact information (address, phone, social media), look for a privacy policy, use a trusted and updated browser.
  • A privacy policy outlines what data a site collects, how it uses it, and how it is kept.

Cookies

  • Small text files placed on a device to store state information — preserve preferences and sign-in information to make browsing easier.
  • HTTP is a stateless protocol — cookies are used to maintain session state across requests.
  • Third-party cookies — set by a domain other than the one you're visiting. Used to track activity across sites for advertising. Avoid these.
  • Best practice: set cookies to delete on exit, block third-party cookies, use browser enhanced tracking protection.

Browser Security

  • Drive-by download — a malicious program installed unintentionally by simply clicking the wrong control.
  • Drive-by mining — cryptomining software installed on your machine that sends results to a malicious party.
  • A secure browser: blocks trackers and third-party cookies, uses blocklists for phishing and malware sites, checks reputation of downloads, restricts risky add-ons and plugins.
  • Private browsing — deletes passwords, search history, and browsing data when the tab or window is closed. Does not make you anonymous.
  • Best practices: avoid storing logins/passwords in the browser, disable location services, auto-clear history on close, keep browser updated.
CH 5

Mobile and Wireless Security

Securing Wi-Fi

  • SSID (Service Set Identifier) — the network name. Broadcast by default (beaconing) so nearby devices can see the network.
  • Disabling SSID broadcast hides the network — users must manually enter the network name and password to connect.
  • Change the router default password immediately — default passwords are publicly listed on sites like routerpasswords.com.
  • Disable remote access on the access point to prevent remote configuration.
  • Activate WPA2 or WPA3 encryption — WPA3 is the strongest as of 2018. Periodically change encryption keys.
  • Disable UPnP (Universal Plug and Play) — designed to help devices find each other, but can be exploited as a vulnerability.

Pre-Shared Keys and Encryption

  • Pre-shared key — all authorised entities share the same key (password or passphrase) to encrypt and secure traffic.
  • Both parties must have the same shared key to encrypt and decrypt data. The key must be kept secret.
  • WPA (Wi-Fi Protected Access) — the most used wireless encryption protocol. Use WPA2 or WPA3 where possible.
  • A passphrase provides stronger security than a password — longer and harder to crack.

MAC Address Filtering

  • Each device has a unique MAC address (12 digits separated by colons) that identifies it on a network.
  • MAC address filtering — configure the router to allow only specific MAC addresses to join the network, restricting access to registered devices only.
  • An access point (WAP) is a station cabled to a LAN that connects wireless devices to an existing network. Each has an SSID.

Mobile Device Threats

  • Threats include phishing/vishing, spoofing, spyware, and connecting to unsecured Wi-Fi networks.
  • Wi-Fi Pineapple — a fake access point set up as an open network trap. Positioned between the victim and the real access point to steal credentials and other information (man-in-the-middle attack). Can be purchased for under $200.
  • Best practices: keep OS and software updated, don't jailbreak your phone, use a password/passphrase/biometric, set an idle timeout, only download apps from trusted sources (Apple iTunes, Google Play, Amazon Appstore), review app permissions before installing, disable geotagging, avoid unsecured networks.
CH 6

Secure Devices and Applications

Encryption

  • Scrambles data into an unreadable format using an algorithm and a key. Plaintext → encrypted → ciphertext. Decryption reverses the process.
  • BitLocker Drive Encryption — encrypts entire drives or logical volumes to protect data at rest.
TypeHow it worksAlgorithmsUse cases
Symmetric Single shared key used by both parties to encrypt and decrypt. Both must have the same key before communication. DES, AES Confidential online communications, HTTPS
Asymmetric Two mathematically related keys. Public key shared with everyone. Private key never shared. Also called public key encryption. RSA, PGP Digital signatures, securely exchanging a secret key, confidentiality
Hybrid Combines both types. Most common in practice. Both Securing and exchanging data in most real-world systems

Hashing

  • A one-way function — takes a variable length input and produces a fixed length output (the message digest). Cannot reverse a hash to recover the original data.
  • Used for: authenticating messages, monitoring data integrity, and storing passwords.
  • Any change in the data changes the hash value — used to detect unauthorised modifications.
  • Testing integrity: store the original hash, then compare with a newly generated hash. If values differ, integrity has been violated.
  • Popular family: SHA (Secure Hash Algorithm). SHA-256 and above are considered secure. SHA-1 is not recommended.

Cryptographic Security Services

ServiceGoalHow it's achieved
ConfidentialityKeep private information privateEncrypt the data
IntegrityEnsure data has not been alteredCreate a hash message digest
AuthenticationConfirm identity of a system objectMAC (Message Authentication Code) — an encrypted message digest
AccountabilityPositively identify an entityDigital signature — signs data using the owner's private key

Applications Using Cryptographic Techniques

  • VPN (Virtual Private Network) — creates a secure encrypted tunnel between two endpoints across an insecure network.
  • HTTPS — encrypts communication between browser and web server using TLS.
  • Blockchain — a shared, immutable ledger for recording transactions using multiple cryptographic techniques.
  • S/MIME (Secure Multipurpose Internet Mail Extensions) — secures email by providing confidentiality, authentication, and integrity.
  • Email threats: malware can be embedded within an email to run malicious code when opened. Also used for spam, spoofing, and phishing attacks. Think before clicking any link or attachment.
