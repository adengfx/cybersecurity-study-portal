OSI Model

Learn about the fundamental networking framework that determines the various stages in which data is handled across a network.

What is the OSI Model?

  • OSI = Open Systems Interconnection Model — a critical framework that dictates how all networking devices send, receive, and interpret data.
  • Main benefit: devices can have different functions and designs on a network while still being able to communicate with each other.
  • Consists of seven layers (Layer 7 to Layer 1). As data passes through each layer, specific information is added — this is called encapsulation.

The seven layers of the OSI model

Layer 1 — Physical

  • References the physical hardware components used in networking — the lowest layer.
  • Devices use electrical signals to transfer data between each other in binary (1s and 0s).
  • Ethernet cables connect devices at this layer.
  • PDU: Bits

Layer 2 — Data Link

  • Focuses on physical addressing of transmission.
  • Receives packets from the Network layer and adds the MAC address of the receiving endpoint.
  • Every network-enabled device has a Network Interface Card (NIC) with a unique MAC address set by the manufacturer — burnt into the card (can be spoofed but not permanently changed).
  • Also responsible for presenting data in a format suitable for transmission.
  • PDU: Frames

Layer 3 — Network

  • Where routing and reassembly of data takes place.
  • Routing determines the most optimal path for data. Factors: shortest path, reliability, physical connection speed (copper vs fibre).
  • Protocols: OSPF (Open Shortest Path First) and RIP (Routing Information Protocol).
  • Everything dealt with via IP addresses at this layer. Routers are Layer 3 devices.
  • PDU: Packets

Layer 4 — Transport

  • Transmits data using either TCP or UDP.

Layer 5 — Session

  • Creates and maintains connections to other devices once data has been formatted by the Presentation layer.
  • Once a connection is established, a session is created — active for the duration of the connection.
  • Responsible for closing connections that are lost or inactive.
  • Sessions can contain checkpoints — if data is lost, only the newest pieces need to be re-sent, saving bandwidth.
  • Sessions are unique — data cannot travel across different sessions.
  • PDU: Data

Layer 6 — Presentation

  • Where standardisation begins. Acts as a translator between the Application layer and the rest of the model.
  • Ensures data sent from one format can be understood by a computer using a different format (e.g. data sent across different email clients).
  • Security features such as data encryption (e.g. HTTPS) occur here.
  • PDU: Data

Layer 7 — Application

  • The layer most familiar to users — where protocols and rules determine how users interact with data.
  • Everyday apps such as browsers and email clients provide a GUI (Graphical User Interface) for users to interact with data sent or received.
  • Other protocols include DNS — how website addresses are translated into IP addresses.
  • PDU: Data
