OSI Model

Learn about the fundamental networking framework — the seven layers that determine how data is handled across a network.

Task 1 — What is the OSI Model?
  • The OSI (Open Systems Interconnection) Model is a critical framework used in networking. It provides a standardised model that dictates how all networking devices send, receive, and interpret data.
  • One of its key benefits is that devices with different functions and designs can still communicate with each other on the same network — because they all follow the same model.
  • The OSI model consists of seven layers, arranged from Layer 7 down to Layer 1. At every individual layer, specific processes take place and specific pieces of information are added to the data being transmitted. This process of adding information at each layer is called encapsulation.

The seven layers of the OSI model

Task 2 — Layer 1: Physical
  • The Physical layer is one of the easiest to grasp. It references the physical hardware components used in networking — the lowest layer of the model.
  • Devices at this layer transfer data between each other using electrical signals in a binary numbering system (1s and 0s).
  • Physical examples include ethernet cables that connect devices together.

Task 3 — Layer 2: Data Link
  • The Data Link layer focuses on physical addressing of transmission. It receives a packet from the Network layer (which includes the IP address of the remote computer) and adds the physical MAC address of the receiving endpoint.
  • Inside every network-enabled device is a Network Interface Card (NIC) — a piece of hardware that comes with a unique MAC address assigned by the manufacturer. This address is literally burnt into the card and cannot be permanently changed (though it can be spoofed).
  • When information is sent across a network, it is the physical MAC address that is used to identify exactly where to deliver it. The Data Link layer is also responsible for presenting data in a format suitable for transmission.

  • What is the name of the piece of hardware that all networked devices come with?

Task 4 — Layer 3: Network
  • The Network layer is where the magic of routing and reassembly of data takes place. Routing determines the most optimal path for data chunks to travel to reach their destination.
  • Protocols at this layer handle path selection: OSPF (Open Shortest Path First) and RIP (Routing Information Protocol). The factors that determine the chosen route are: which path is shortest (fewest devices to traverse), which path is most reliable (have packets been lost on it before), and which path has the fastest physical connection (copper is slower, fibre is considerably faster).
  • Everything at this layer is dealt with via IP addresses. Devices capable of delivering packets using the Network layer — such as routers — are known as Layer 3 devices.

Task 5 — Layer 4: Transport
  • The Transport layer plays a vital part in transmitting data across a network. When data is sent between devices it follows either TCP or UDP.
  • TCP (Transmission Control Protocol) is designed with reliability and guarantee in mind. It reserves a constant connection between two devices for the duration of the transfer and incorporates error checking — ensuring data sent in small chunks from the Session layer has been received and reassembled in the correct order. Advantages: guarantees accuracy of data, capable of synchronising between devices to prevent flooding. Disadvantage: significantly slower than UDP due to the additional processing involved. Used for file sharing, internet browsing, and sending email — situations where data must be accurate and complete.
  • UDP (User Datagram Protocol) is not nearly as advanced. Any data sent via UDP is sent regardless of whether it arrives — there is no synchronisation, no error checking, and no guarantee. Advantages: much faster than TCP, leaves the application layer to decide pacing, does not reserve a continuous connection. Disadvantage: doesn't care if data is received, meaning unstable connections result in a poor user experience. UDP is useful for video streaming (where a few pixelated frames are acceptable) and for protocols like ARP and DHCP.

TCP — reliable and connection-based but slower

UDP — fast and stateless but no delivery guarantee

Task 6 — Layer 5: Session
  • Once data has been correctly translated and formatted from the Presentation layer, the Session layer begins to create and maintain a connection to the destination device.
  • Once a connection is established, a session is created — and it remains active for as long as the connection is active. The Session layer is also responsible for closing connections that have been lost or left idle.
  • Sessions can contain checkpoints — if data is lost partway through, only the most recent pieces need to be re-sent rather than starting over, which saves bandwidth.
  • Sessions are unique — data cannot travel across different sessions, only within its own.

  • What is the name of this layer?

Task 7 — Layer 6: Presentation
  • The Presentation layer is where standardisation begins. Software can be developed in many different ways, but data ultimately needs to be handled in a consistent way — this layer acts as a translator between the Application layer and the rest of the model.
  • It ensures that data sent from one software format can be understood by a receiving device using a different format — for example, data sent across different email clients.
  • Security features such as data encryption (e.g. HTTPS) occur at the Presentation layer.
Answer the questions below
  • What is the name of this layer?
    Presentation
  • What is the main purpose that this layer acts as?
    Translator
Task 8 — Layer 7: Application
  • The Application layer is the layer most familiar to everyday users. It is where protocols and rules are in place to determine how users should interact with data being sent or received.
  • Everyday applications such as browsers and email clients provide a GUI (Graphical User Interface) for users to interact with data at this layer.
  • Other protocols operating at the Application layer include DNS — the system that translates human-readable website addresses into IP addresses that computers can understand.

    Application
  • What is the technical term given to the name of the software that users interact with?
