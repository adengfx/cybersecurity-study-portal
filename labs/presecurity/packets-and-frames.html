Packets & Frames

Understand how data is divided into smaller pieces and transmitted across a network — packets, frames, TCP/IP, the three-way handshake, UDP, and ports.

Packets and Frames

  • Packet: A piece of data from Layer 3 (Network) — contains IP header and payload. Used when talking about IP addresses.
  • Frame: Used at Layer 2 (Data Link) — encapsulates the packet and adds additional info such as MAC address.
  • Analogy: the frame is the envelope, the packet is the letter inside. Once the recipient opens the envelope (frame) they know how to forward the letter (packet).
  • Packets are an efficient way of communicating data — small pieces reduce the chance of bottlenecks compared to sending large messages at once.
  • This process of wrapping data with additional information at each layer is called encapsulation.

IP Packet Headers

HeaderPurpose
Time to Live (TTL)Sets an expiry timer for the packet so it doesn't clog the network if it never reaches its destination.
ChecksumProvides integrity checking — if any data has changed, the checksum will be different and the packet discarded.
Source addressIP address of the device sending the packet — so data knows where to return to.
Destination addressIP address of the device receiving the packet — so data knows where to travel next.

TCP/IP Model

  • TCP/IP consists of 4 layers — an arguably simplified version of the OSI model.
  • Layers: Application, Transport, Internet, Network Interface.
  • Information is added to each layer of the TCP model as the packet traverses it — similar to OSI encapsulation.
  • TCP is connection-based — must establish a connection before any data is sent. This guarantees data sent will be received on the other end.

TCP Three-Way Handshake

  • TCP must establish a connection before transmitting data — this is the three-way handshake.
  • Any sent data is given a random sequence number and reconstructed using that number, incrementing by 1. Both computers agree on the order during the handshake.
STEP 1
SYN
Client sends initial packet to initiate connection and synchronise sequence numbers.
STEP 2
SYN/ACK
Server acknowledges the SYN attempt and sends its own ISN back to the client.
STEP 3
ACK
Client acknowledges the server's ISN — connection is now established.
STEP 4
DATA
Data (e.g. bytes of a file) is transmitted across the established connection.
STEP 5
FIN
Used to cleanly close the connection once all data has been received.
RST
Reset
Abruptly ends all communication — a last resort indicating a problem in the process.

UDP (User Datagram Protocol)

  • Unlike TCP, UDP is stateless — no connection required before sending data, no three-way handshake, no synchronisation.
  • Used where applications can tolerate data loss (video streaming, voice chat) or where an unstable connection is expected.
  • UDP packets are simpler than TCP — fewer headers. No checksum for data integrity.
  • Advantages: Much faster than TCP; leaves the application layer to control packet pacing; doesn't reserve a continuous connection.
  • Disadvantages: Doesn't care if data is received; unstable connections result in poor user experience.

Ports

  • Ports are the points through which data is exchanged — like a harbour where ships (data) dock at specific berths (ports).
  • In computing, ports are numerical values between 0–65535.
  • Ports 0–1024 are known as common ports — standardised for specific protocols.
ProtocolPortPurpose
FTP21File Transfer Protocol — client-server file sharing.
SSH22Secure Shell — securely login to systems via text-based interface.
HTTP80HyperText Transfer Protocol — powers the World Wide Web.
HTTPS443Secure HTTP — encrypted version using TLS/SSL.
SMB445Server Message Block — file and device (e.g. printer) sharing.
RDP3389Remote Desktop Protocol — virtual desktop access to a system.
