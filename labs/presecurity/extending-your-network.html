Extending Your Network

Learn about the technologies used to extend networks — port forwarding, firewalls, VPNs, and Layer 2/3 switches.

Task 1 — Introduction to Port Forwarding
  • Port forwarding is an essential component for connecting applications and services to the internet. Without it, web servers and other services are only accessible to devices within the same local network — this internal-only access is known as an intranet.
  • If an admin wants a service to be publicly accessible, they must implement port forwarding. This opens specific ports on the router, allowing external traffic from the internet to reach the internal server.
  • Port forwarding is easy to confuse with firewall behaviour. Port forwarding opens specific ports; firewalls determine whether traffic is permitted to travel across those ports. Even if ports are opened via port forwarding, a firewall can still block traffic on them.

Task 2 — Firewalls 101
  • A firewall is a device within a network responsible for determining what traffic is allowed to enter or exit — think of it as border security for a network.
  • An admin can configure a firewall to permit or deny traffic based on: where traffic is coming from, where it is going, which port it is destined for, and which protocol (TCP, UDP, or both) it uses. Firewalls perform packet inspection to determine answers to these questions.
  • Firewalls come in all shapes and sizes — from dedicated hardware appliances in large enterprise networks to software-based firewalls like Snort on a home system.
  • Stateful firewall: Uses the entire connection context (not just individual packets) to make dynamic decisions about whether traffic is acceptable. More resource-intensive but more intelligent — for example, if a connection from a host is determined to be bad, the entire device can be blocked. Can allow the early stages of a TCP handshake that later fails.
  • Stateless firewall: Applies a static set of rules to evaluate individual packets in isolation. A device sending one bad packet will not necessarily mean the entire device is blocked. Much less resource-intensive but less intelligent — only as effective as the rules defined. Particularly useful against large volumes of traffic from a fixed set of hosts, such as a Distributed Denial of Service (DDoS) attack.
Answer the questions below
  • What is the technical term for the type of firewall that inspects the entire connection?
    Stateful
  • What is the technical term for the type of firewall that inspects individual packets?
    Stateless

Task 4 — VPN Basics
  • A VPN (Virtual Private Network) is a technology that allows devices on separate, geographically different networks to communicate securely by creating a dedicated encrypted path between them — known as a tunnel. Devices connected through this tunnel form their own private network.
  • For example, two offices in different cities can be connected via VPN so that resources like servers and infrastructure can be accessed from either location as if they were on the same LAN.
  • VPN benefits: connectivity (geographically separate networks can communicate), privacy (encryption protects data in transit — useful on public Wi-Fi), anonymity (journalists and activists depend on VPNs to safely report in countries with restricted freedom of speech).
  • PPP: Technology used by PPTP to allow authentication and encrypt data. Uses a private key and public certificate (similar to SSH). Not capable of leaving a network by itself (non-routable).
  • PPTP (Point-to-Point Tunneling Protocol): Allows PPP data to travel and leave a network. Very easy to set up and supported by most devices, but weakly encrypted compared to alternatives.
  • IPSec (Internet Protocol Security): Encrypts data using the existing IP framework. Difficult to set up but provides strong encryption and is supported on many devices.

Task 5 — LAN Networking Devices
  • A router's job is to connect networks and pass data between them — this is called routing. Routers operate at Layer 3 of the OSI model and often feature an admin interface (web or console) for configuring port forwarding and firewall rules. When devices are connected by multiple paths, routing protocols decide which path to use based on: shortest path, reliability, and physical connection speed (copper vs fibre).
  • A switch is a dedicated networking device responsible for connecting multiple devices using ethernet. Switches can operate at either Layer 2 or Layer 3 of the OSI model — these are exclusive; a Layer 2 switch cannot also operate at Layer 3.
  • Layer 2 switch: Forwards frames to the correct device using MAC addresses. Solely responsible for getting the frame to the right device on the same network — cannot route between networks.
  • Layer 3 switch: More sophisticated — can perform some routing responsibilities in addition to switching. Can forward frames (as Layer 2 does) and also route packets between networks using IP.
  • VLAN (Virtual Local Area Network): Allows specific devices within a network to be virtually split into separate segments, even if they are physically connected to the same switch. Each segment is treated as its own network with its own rules — providing security through isolation. Example: a Sales department and an Accounts department can both access the internet via the same switch but cannot communicate directly with each other.
