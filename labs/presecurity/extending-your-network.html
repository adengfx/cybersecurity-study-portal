Extending Your Network

Learn about the technologies used to extend networks onto the internet — port forwarding, firewalls, VPNs, and Layer 2/3 switches.

Port Forwarding

  • Essential component for connecting applications and services to the internet.
  • Without port forwarding, servers are only accessible to devices within the same network (intranet).
  • Port forwarding opens specific ports on a router so that traffic from the internet can reach internal servers.
  • Easy to confuse with firewalls — port forwarding opens ports; firewalls determine if traffic can travel across those ports. Even if ports are open via port forwarding, a firewall can still block traffic.

Without port forwarding — server only accessible within the local network (intranet)

With port forwarding — server accessible from the public internet

Firewalls

  • A device within a network responsible for determining what traffic is allowed to enter or exit — like border security for a network.
  • An admin can configure rules based on: source network, destination network, destination port, and protocol (TCP/UDP/both).
  • Firewalls perform packet inspection to determine whether traffic meets the configured rules.
  • Come in all shapes and sizes — dedicated hardware (large networks) or software-based (home networks).
TypeHow it works
StatefulUses the entire connection context rather than inspecting individual packets. Dynamically determines behaviour based on the state of the connection. Resource-intensive but more intelligent.
StatelessUses a static set of rules to determine whether individual packets are acceptable or not. Less resource-intensive but less intelligent — only as effective as the rules defined.

VPN Basics

  • A Virtual Private Network allows devices on separate networks to communicate securely by creating a dedicated tunnel between them.
  • Devices on a VPN are still part of their original networks but also form a private network together (e.g. two offices in different countries communicating as if on the same LAN).
BenefitDescription
ConnectivityNetworks in different geographical locations can be connected — useful for businesses with multiple offices.
PrivacyVPN uses encryption to protect data — only understandable between the sending and receiving devices.
AnonymityJournalists and activists rely on VPNs to safely report in countries with restricted freedom of speech.
VPN TechnologyDescription
PPPUsed by PPTP to allow authentication and encrypt data. VPNs work by using a private key and public certificate — similar to SSH.
PPTPPoint-to-Point Tunneling Protocol — allows PPP data to travel and leave a network. Easy to set up but weakly encrypted compared to alternatives.
IPSecInternet Protocol Security — encrypts data using the existing IP framework. Difficult to set up but strongly encrypted and supported by many devices.

LAN Networking Devices — Deeper Dive

  • Routers operate at Layer 3 of OSI. Often feature an admin interface (web/console) for configuring port forwarding and firewall rules.
  • When multiple paths exist, routing protocols decide which to use based on: shortest path, reliability, and connection speed.

Router selecting the most optimal path between networks

Switch TypeOSI LayerFunction
Layer 2 SwitchLayer 2 (Data Link)Forwards frames to the correct device based on MAC address. Solely responsible for same-network delivery — cannot route between networks.
Layer 3 SwitchLayer 3 (Network)More sophisticated — can perform some routing responsibilities in addition to switching. Can route between VLANs.

Layer 2 switch — forwards frames based on MAC addresses

  • VLAN (Virtual Local Area Network) — allows specific devices within a network to be virtually split into separate segments.
  • All VLAN devices can still benefit from shared resources (e.g. internet) but are treated separately — providing network security through segregation.
  • Example: Sales and Accounts departments can both access the internet but cannot communicate directly with each other.
