Intro to LAN

Learn about the technologies and designs that power private networks — topologies, switches, routers, subnetting, ARP, and DHCP.

Task 1 — Introducing LAN Topologies

Star topology: devices individually connected via a central switch/hub. Most common today — scalable and robust but expensive and fails if central device fails. Bus topology: all devices stem from a single backbone cable — easy and cost-efficient but prone to bottlenecks and single point of failure. Ring (token) topology: devices connected in a loop, data travels one direction — little cabling needed but a cut cable breaks the entire network. Switches aggregate devices, track which device is on each port, and send packets directly rather than broadcasting. Routers connect networks and pass data between them via routing — creating paths for data delivery.

Task 2 — A Primer on Subnetting

Subnetting splits a network into smaller networks using a subnet mask (32-bit, four octets, ranging 0–255). Subnets use three address types: Network address (identifies start of network, e.g. 192.168.1.0), Host address (identifies a specific device, e.g. 192.168.1.100), Default gateway (device capable of sending data to another network, usually .1 or .254). Benefits: efficiency, security, full control.

  • What is the technical term for dividing a network up into smaller pieces?
    Subnetting
  • How many bits are in a subnet mask?
    32
  • What is the range of a section (octet) of a subnet mask?
    0–255
  • What address is used to identify the start of a network?
    Network Address
  • What address is used to identify devices within a network?
    Host Address
  • What is the name used to identify the device responsible for sending data to another network?
    Default Gateway
Task 3 — ARP (Address Resolution Protocol)

ARP allows devices to associate a MAC address with an IP address on a network. Each device maintains an ARP cache — a ledger of MAC-to-IP mappings. ARP Request: broadcast to entire network asking which device owns a specific IP. ARP Reply: device with matching IP responds with its MAC address, which is stored in the requester's cache.

ARP request and reply process

  • What does ARP stand for?
    Address Resolution Protocol
  • What category of ARP packet asks a device whether or not it has a specific IP address?
    Request
  • What address is used as a physical identifier for a device on a network?
    MAC Address
  • What address is used as a logical identifier for a device on a network?
    IP Address
Task 4 — DHCP

IP addresses can be assigned manually or automatically via DHCP (Dynamic Host Configuration Protocol). The four-step process: DHCP Discover (device broadcasts to find a DHCP server) → DHCP Offer (server offers an IP address) → DHCP Request (device confirms it wants the offered IP) → DHCP Ack (server confirms assignment is complete).

DHCP four-step process

  • What type of DHCP packet is used by a device to retrieve an IP address?
    DHCP Discover
  • What type of DHCP packet does a device send once it has been offered an IP address?
    DHCP Request
  • Finally, what is the last DHCP packet sent to a device from a DHCP server?
    DHCP Ack
