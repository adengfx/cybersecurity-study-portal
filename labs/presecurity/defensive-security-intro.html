Defensive Security Intro

Introducing defensive security — Threat Intelligence, SOC, DFIR, Malware Analysis, and SIEM.

Task 1 — Introduction to Defensive Security
  • Defensive security is concerned with two main tasks: preventing intrusions from occurring, and detecting and responding to intrusions when they do occur.
  • Blue teams form the core of the defensive security landscape.
  • Key defensive tasks include: user awareness training, documenting and managing assets, updating and patching systems, setting up firewalls and IPS (Intrusion Prevention Systems), and implementing proper logging and monitoring.
  • Defensive security also encompasses: Security Operations Centers (SOC), Threat Intelligence, Digital Forensics and Incident Response (DFIR), and Malware Analysis.
Answer the questions below
  • Which team focuses on defensive security?
    Blue Team
Task 2 — Areas of Defensive Security
  • Security Operations Center (SOC) — a team of cybersecurity professionals who monitor a network and its systems to detect malicious events. Main areas of interest: vulnerabilities (weaknesses requiring patching or mitigation), policy violations (e.g. uploading company data to an unauthorised service), unauthorised activity (e.g. stolen credentials being used), and network intrusions.
  • Threat Intelligence — information gathered about actual and potential enemies. Purpose is threat-informed defence. Data is collected (locally via network logs or publicly via forums/feeds), processed into a usable format, and analysed to identify attacker TTPs (Tactics, Techniques and Procedures) and predict future activity.
  • Digital Forensics — applies scientific methods to investigate digital crimes. Focus areas: file system (installed programs, created/deleted files), system memory (malware that runs only in memory), system logs (records of activity even after an attacker attempts to clear traces), and network logs.
  • Incident Response — a structured methodology for responding to data breaches and cyber attacks. Four phases: Preparation → Detection and Analysis → Containment, Eradication and Recovery → Post-Incident Activity.
  • Malware Analysis — studying malicious software to understand its purpose and behaviour. Static analysis inspects the program without running it (requires assembly language knowledge). Dynamic analysis runs it in a controlled environment to observe its behaviour.
  • Malware types: Virus (attaches to programs, spreads and damages files), Ransomware (encrypts files and demands payment for decryption), Trojan horse (disguises itself as legitimate software while hiding a malicious function).
Answer the questions below
  • What would you call a team of cyber security professionals that monitors a network and its systems for malicious events?
    Security Operations Center
  • What does DFIR stand for?
    Digital Forensics and Incident Response
  • Which kind of malware requires the user to pay money to regain access to their files?
    Ransomware
Task 3 — Practical Example of Defensive Security
  • Simulated working as a SOC analyst at a bank. The bank's SOC uses a SIEM (Security Information and Event Management) tool, which gathers security-related events from various sources and presents them in a dashboard, generating alerts when suspicious activity is detected.
  • Not all alerts are malicious — for example, multiple failed login attempts may simply mean a user has forgotten their password. Unknown IP connections could be a new legitimate user or an unauthorised access attempt. The analyst uses expertise to distinguish between the two.
  • Workflow followed: SIEM alert received → identified potential malicious attempt → investigated the IP address using an IP scanner → scanner confirmed the IP was malicious → escalated to team lead → IP address blocked.
