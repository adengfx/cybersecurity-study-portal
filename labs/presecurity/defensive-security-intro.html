Defensive Security Intro

Introducing defensive security and related topics — Threat Intelligence, SOC, DFIR, Malware Analysis, and SIEM.

What is Defensive Security?

  • Concerned with two main tasks: preventing intrusions from occurring and detecting and responding to intrusions when they do occur.
  • Blue teams are the core of the defensive security landscape.
TaskDescription
User awarenessTraining users about cybersecurity helps protect against attacks targeting human error.
Asset managementDocumenting and managing systems and devices so everything that needs protecting is known.
PatchingEnsuring computers, servers, and network devices are correctly updated against known vulnerabilities.
Preventative devicesFirewalls control network traffic; Intrusion Prevention Systems (IPS) block malicious traffic.
Logging & monitoringEssential for detecting malicious activity — e.g. new unauthorised devices appearing on the network.

Security Operations Center (SOC)

  • Team of cybersecurity professionals that monitor a network and its systems to detect malicious events.
  • Vulnerabilities: Weaknesses in systems — SOC tracks and ensures they are patched or mitigated.
  • Policy violations: E.g. a user uploading confidential data to an unauthorised cloud storage service.
  • Unauthorised activity: Detecting when stolen credentials are used to log in to the network.
  • Network intrusions: Detecting intrusions as quickly as possible — can occur via malicious links or exploitation of public servers.

Threat Intelligence

  • Intelligence = information gathered about actual and potential enemies. A threat is any action that can disrupt or affect a system.
  • Purpose: achieve threat-informed defence. Different companies face different adversaries (e.g. nation-state actors vs. ransomware groups).
  • Data collection: Locally via network logs, or publicly via forums and threat feeds.
  • Data processing: Arranges raw data into a format suitable for analysis.
  • Analysis: Finds information about attacks and motives, produces actionable recommendations, identifies TTPs (Tactics, Techniques & Procedures) of threat actors.

Digital Forensics

  • Application of science to investigate crimes and establish facts using digital evidence.
  • File system: Forensic image of storage reveals installed programs, created/deleted files.
  • System memory: If malware runs in memory without saving to disk, a forensic memory image is the best way to analyse it.
  • System logs: Even if an attacker clears their traces, some log entries will remain.
  • Network logs: Logs of network packets help answer whether an attack is occurring and what it entails.

Incident Response — Four Phases

PHASE 1
Preparation
Team trained and ready to handle incidents. Preventative measures in place.
PHASE 2
Detection & Analysis
Necessary resources to detect incidents and analyse their severity.
PHASE 3
Containment, Eradication & Recovery
Stop the spread, eliminate the threat, restore affected systems.
PHASE 4
Post-Incident Activity
Report produced, lessons learned shared to prevent similar future incidents.

Malware Analysis

  • Malware = malicious software.
  • Virus: Attaches to a program and spreads computer to computer by altering, overwriting, or deleting files.
  • Ransomware: Encrypts user files, making them unreadable. Attacker demands payment for the decryption key.
  • Trojan horse: Appears to perform a desirable function but hides a malicious payload underneath.
  • Static analysis: Inspects the malicious program without running it — requires knowledge of assembly language.
  • Dynamic analysis: Runs the malware in a controlled environment and monitors its behaviour.

SIEM in Practice

  • SIEM (Security Information and Event Management) — gathers security-related info and events from various sources into a centralised dashboard.
  • Not all alerts are malicious — the analyst uses expertise to determine which ones are harmful.
  • SOC workflow: review alert → investigate IP in threat intelligence scanner → escalate to team lead → block IP address.
