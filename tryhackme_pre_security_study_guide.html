Offensive vs Defensive Security

TypeGoalKey Teams
OffensiveBreak into systems (with authorisation) to find vulnerabilities before attackers doRed Team, Pen Testers
DefensivePrevent intrusions, detect them when they occur, and respond effectivelyBlue Team, SOC Analysts

Defensive Security Areas

  • SOC — monitors networks for vulnerabilities, policy violations, unauthorised activity, and intrusions.
  • Threat Intelligence — collects and analyses data about adversaries to achieve threat-informed defence. Identifies TTPs.
  • DFIR — investigates file systems, system memory, system logs, and network logs to establish facts about incidents.
  • Malware Analysis — static analysis (inspect without running) and dynamic analysis (run in controlled environment).
  • SIEM — aggregates security events into a centralised dashboard for analyst review.

What is Networking?

What is Networking? →

Key Concepts

  • The Internet is a network of networks — private networks connected via public networks. Invented as we know it in 1989 by Tim Berners-Lee (World Wide Web). First documented network: ARPANET (late 1960s, US Dept of Defense).
  • IP address = name. Divided into four octets. Public (internet-facing) or private (local network). IPv4 exhausted — IPv6 supports 340 trillion+ addresses.
  • MAC address = fingerprint. 12-character hex. First half: vendor. Second half: unique to device. Can be spoofed.
  • Ping uses ICMP to test connection performance via echo packets. Syntax: ping <IP/URL>

Switches, Routers, Subnetting, ARP & DHCP

  • Switch: Aggregates devices, tracks which device is on each port, sends packets directly. Available 4–64 ports.
  • Router: Connects networks via routing (Layer 3 OSI). Chooses optimal path by shortest, reliability, speed.
  • Subnetting: Splits network into smaller segments using a 32-bit subnet mask. Three address types: Network (192.168.1.0), Host (192.168.1.100), Default gateway (192.168.1.1).
  • ARP: Maps MAC address to IP address. ARP Request (broadcast) → ARP Reply (device responds with MAC) → stored in ARP cache.
  • DHCP: Auto-assigns IPs. Process: Discover → Offer → Request → Ack.

Packets & Frames

Packets & Frames →

Key Concepts

  • Packet (Layer 3) — contains IP header and payload. Frame (Layer 2) — encapsulates packet and adds MAC address. Frame = envelope, packet = letter.
  • Notable IP headers: TTL (expiry timer), Checksum (integrity), Source address, Destination address.
  • TCP/IP model — 4 layers: Application, Transport, Internet, Network Interface.
  • Three-way handshake: SYN → SYN/ACK → ACK → DATA → FIN. RST = abrupt termination.
  • UDP — stateless, no handshake, no guaranteed delivery. Simpler headers than TCP.

Extending Your Network

Extending Your Network →

Port Forwarding, Firewalls, VPNs & Switches

  • Port forwarding — opens ports on a router so internet traffic can reach internal servers. Port forwarding opens; firewalls control.
  • Stateful firewall — uses full connection context for dynamic decisions. Resource-intensive but intelligent.
  • Stateless firewall — applies static rules to individual packets. Less resource-intensive but only as good as rules defined.
  • VPN — creates encrypted tunnel between devices on separate networks. Technologies: PPP (auth + encryption, non-routable), PPTP (allows PPP to leave network, weakly encrypted), IPSec (strong encryption, uses IP framework).
  • Layer 2 switch — forwards frames by MAC address only. Layer 3 switch — can also route packets via IP. Exclusive — cannot do both.
  • VLAN — virtually splits devices into separate segments on same physical infrastructure for security isolation.
